프로젝트

기업의 인프라 구축

개요

기업의 인프라 구조를 이해하고 직접 설계 및 구축을 해보며 보안적인 문제점들을 보완할 보안 대책 강구

요약

  • LAN, WAN, DMZ 영역을 고려하며 토폴로지 작성 및 계획 수립

  • 리눅스 환경의 가상 머신에서 DNS, DHCP, CA, DB, VPN, WEB 그리고 모니터링 서버 등 기본적인 서버를 구축

  • 서버들 사이의 안전한 네트워크 통신을 위해 NAT와 ACL 기능을 탑재한 라우터와 방화벽을 구축하여 각 서버 연결

  • Bash 스크립트와 Crontab을 이용한 베이터베이스 백업 환경 구축

  • 시스템, 웹 취약점 점검 후 문제점을 보완하고 안전한 기업 인프라 환경을 구축하기 위한 보안 대책 강구

PPT 다운로드

뮤직 컨트롤러

개요

개발부터 배포까지의 진행 과정 이해를 위한 Django와 React를 이용한 웹 애플리케이션 개발 및 AWS와 Docker를 활용한 배포

요약

  • 웹 프레임워크인 Djacgo와 React를 활용한 웹 애플리케이션 개발

  • 사용자 인증과 사용자의 음악 재생을 제어하기 위한 Spotify API를 활용

  • dockerfile과 docker-compose.yml 파일을 작성하여 Docker 컨테이너 환경에서 웹 애플리케이션을 리버스 프록시와 함께 실행하고 AWS EC2 인스턴스를 활용하여 배포

  • 인증되지 않은 Spotify 계정으로는 방 생성이 어렵습니다. 제 룸 코드를 사용하여 제 방으로 입장해 주세요, "CRXVQI"

더보기 사이트 방문

웹 프로파일

개요

나에 대한 정보에 쉽게 접근할 수 있는 웹사이트 개발 및 Nginx 리버스 프록시를 활용하여 여러 컨테이너 서버들 간의 요청 처리 기술 이해

요약

  • HTLM, CSS, Javascript, 그리고 Jquery를 활용한 웹사이트 개발

  • Nginx 컨테이너를 활용하여 리버스 프록시 서버 및 웹 서버를 구축하고 클라이언트의 요청을 알맞은 서버로 전송

  • AWS의 Route53을 활용하여 개인 도메인과 AWS EC2 인스턴스를 매핑하여 온라인상에 배포

  • 웹사이트로 들어오는 요청에 대한 보안 강화를 위해 "letsencrypt"에서 발급받은 ssl 인증서를 통해 HTTPS 프로토콜 설정
